The naive approach

2023-03-09

We recently did the famous bowling kata and I was reminded of only writing the simplest thing that would make a test pass. This is at times also referred to as the naive approach.

This is something that we not only do for code but also for other levels of abstraction and infrastructure. For instance we start with vanilla js until we see a clear need for some sort of template engine and other frontend frameworks. If our product can just store a file with the data we need we use that until we see a clear need for a more proper database and so on.
