Extract to Cloud Run

2023-02-18

Back in resharper days there was a refactoring method called "extract to method". I recently realized that a quite common pattern I'm using today could be labeled "extract to cloud run".

The context is that you have some form of long running app running in your k8s cluster. Certain operations in the app have a much higher demand on the computer resources (memory and/or cpu) that are not needed when that operation isn't running. What we typically do in situations like these is moving that operation to cloud run. The cloud run can be beefed up as needed and the long running app can have it's resources turned down to the minimum it needs to operate.

All in all this leads to happier k8s orchestration and you can beef up the cloud run get the computer resource intense operation to complete quickly and only pay the high resource price when needed.
